# Задание: Панель управления серверами и камерами в Docker

Этот архив содержит формулировку задания для бэкенд- и фронтенд-разработчиков, а также тестовые данные.

Тематика: система управления серверами и IP-камерами, запущенными в Docker-контейнерах.

## Общая идея

Представьте, что у компании есть сотни серверов с Docker и тысячи камер видеонаблюдения, каждая камера — это контейнер с видеопотоком. Нужна внутренняя панель администратора, которая позволяет:

- управлять списком **серверов** (CRUD);
- управлять списком **камер** и их привязкой к серверам (CRUD);
- запускать типовые операции над камерами и серверами (старт/стоп/рестарт);
- загружать и очищать данные по камерам из большого CSV-файла (≈40 000 строк);
- просматривать состояние инфраструктуры через удобный веб-интерфейс (desktop + mobile).

В архиве вы найдёте:

- `backend/BACKEND_TASK.md` — требования к бэкенду (индустриальные);
- `backend/openapi.yaml` — спецификация REST API;
- `frontend/FRONTEND_TASK.md` — требования к фронтенду;
- `data/README_DATASET.md` — описание датасета;
- `data/cameras_raw.csv` — «грязный» CSV с камерами (~40k строк);
- `tests/backend.postman_collection.json` — коллекция запросов для проверки API;
- `tests/frontend_manual_checklist.md` — чек-лист для ручного тестирования фронтенда.

Рекомендуемый способ использования:

1. Дать разработчикам этот архив как постановку задачи.
2. Оценивать решения по:
   - полноте реализации требований;
   - качеству кода и архитектуры;
   - корректности работы с данными из CSV;
   - стабильности и UX фронтенда (включая мобильную версию);
   - прохождению тестов/сценариев из папки `tests`.
