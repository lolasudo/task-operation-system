# Чек-лист для ручного тестирования фронтенда

## 1. Общие проверки

- [ ] Приложение стартует без ошибок в консоли браузера.
- [ ] Подключение к бэкенду настраивается через переменную окружения/конфигурацию.
- [ ] При отсутствии бэкенда пользователь видит понятное сообщение об ошибке.

## 2. Авторизация

- [ ] При открытии приложения без токена показывается экран логина.
- [ ] Ввод корректных учётных данных приводит к переходу в основное приложение.
- [ ] При обновлении страницы с защищёнными маршрутами пользователь остаётся авторизованным (если токен валиден).
- [ ] При истечении/удалении токена пользователь перенаправляется на экран логина.

## 3. Список серверов

- [ ] Отображается таблица/список серверов.
- [ ] Работает пагинация (переключение страниц).
- [ ] Фильтр по статусу меняет набор отображаемых серверов.
- [ ] Поиск по имени/hostname работает (находит ожидаемые записи).
- [ ] Кнопка «Перезапустить» сервер вызывает соответствующий endpoint (по сети видно запрос).

## 4. Карточка сервера

- [ ] Открывается при клике на сервер из списка.
- [ ] Отображаются детали сервера (hostname, IP, статус, локация и т.п.).
- [ ] Изменение полей и сохранение вызывает соответствующий запрос к API.
- [ ] При ошибке сохранения показывается сообщение об ошибке.

## 5. Список камер

- [ ] Отображается список камер с ключевыми полями.
- [ ] Работают фильтры по статусу, серверу, активности.
- [ ] Поиск по имени/IP/RTSP-URL работает.
- [ ] Пагинация переключает страницы.

## 6. Карточка камеры

- [ ] Открывается при клике на камеру.
- [ ] Отображаются детали камеры.
- [ ] Редактирование и сохранение полей работает (видно запросы к API).
- [ ] Смена сервера для камеры возможна (если предусмотрено API).

## 7. Операции Start/Stop/Restart

- [ ] Кнопки операций отображаются в списке камер и/или карточке камеры.
- [ ] По нажатию отправляется запрос на соответствующий endpoint.
- [ ] Пользователь получает визуальную обратную связь (лоадер/статус/уведомление).
- [ ] При ошибке отображается сообщение об ошибке.

## 8. Импорт CSV

- [ ] На странице импорта есть элемент выбора файла.
- [ ] При выборе и отправке файла выполняется запрос к бэкенду.
- [ ] Отображается статус импорта (успех/ошибка).
- [ ] При успехе отображается краткая статистика (сколько строк загружено, сколько отклонено).
- [ ] При ошибке отображается понятное сообщение.

## 9. Адаптивность

### Desktop

- [ ] На широком экране (≥1280px) таблицы занимают большую часть ширины.
- [ ] Нет горизонтального скролла всей страницы (только таблицы при необходимости).

### Mobile

- [ ] Интерфейс корректно отображается на ширине ~360–414px.
- [ ] Основные действия доступны (авторизация, просмотр списков, детали).
- [ ] Таблицы не «ломают» верстку, при необходимости превращаются в карточки.
- [ ] Элементы управления (кнопки, ссылки) достаточно крупные для тач-управления.

## 10. Ошибки и стабильность

- [ ] При сетевых ошибках (отключение бэкенда) интерфейс не «ломается», а показывает понятное сообщение.
- [ ] Не возникает бесконечных спиннеров без объяснения причин.
- [ ] При ошибках API (4xx/5xx) ошибки корректно перехватываются и отображаются пользователю.
